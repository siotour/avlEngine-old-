avl::utility::Image::Image ( const std::string &  file_name)

Attempts to load an image given only a file name.

file_name must have the extension of a currently implemented image file format, and the data contained in the file must match that file format (i.e. you will run into problems if you simply rename a .PNG file to .TGA and then try loading it).

Postcondition:
If an error occurs while reading from the file, if the file's formatting is not supported, or if an unsupported file extension is supplied, the created Image will have 0 width, height, pixel_depth, and pixel_data pointer will be NULL.
See the image-loading functions for each individual image file format to see what additional restrictions are imposed for each file format.
